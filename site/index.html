patient, grounded, immovably strong

An agentic AI framework where every capability is a filesystem path. Built on StructFS. Read a path to inspect state. Write a path to change it.

The ox walks

ox is a composable agent runtime where system prompts, conversation history, tool definitions, and model configuration all live as paths in a virtual filesystem. Read a path to inspect state. Write a path to change it. The namespace is the agent.

Tools are shell-provided, not baked in. The kernel runs an agentic loop — reading a prompt from the namespace, streaming a completion, writing results back, repeating until the model stops calling tools. Every shell (browser, WASI, native) brings its own tools and transport.

Architecture

ox-kernel
core
State machine, Tool and Transport traits, ToolRegistry, agentic loop, StructFS re-exports. The bedrock everything rests on.
ox-context
namespace
Namespace store router, SystemProvider, ToolsProvider, ModelProvider. Prompt synthesis — read /prompt and get a complete request.
ox-history
memory
HistoryProvider with Reader/Writer impl. Conversation state as a StructFS store you can read and append to.
ox-core
agent
Agent composition — wires kernel, namespace, and stores together. Re-exports all public types. The yoke that connects everything.
ox-web
browser
WASM shell with JS tool registration, theme picker, and debug UI. The playground where you can talk to the agent in your browser.
ox-dev-server
proxy
Axum binary that proxies to the Anthropic API. Serves the WASM playground and handles API key management.
ox-wasi
stub
WASI target shell. Re-exports ox-core for server-side WebAssembly environments.
ox-emscripten
stub
Emscripten target shell. Re-exports ox-core for legacy WASM compilation pipelines.
